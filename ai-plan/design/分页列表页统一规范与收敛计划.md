# 分页列表页统一规范与收敛计划

本文档是 `web` 分页 / 列表页布局治理的设计真值补充，目标不是重做页面，而是建立可复用、可执行的页面结构规范，并为后续逐页收敛提供迁移顺序。

适用范围：

- `web/src/modules/**` 下的分页列表页和典型列表管理页
- `web/src/shared/**`、`web/src/components/**` 中与分页、筛选、表格、列设置相关的共享组件
- `list-form-detail`、`query-builder-list-detail`、`log-audit` 页面类型

壳层宽度 authority：

- `web/src/utils/route/meta.ts` 根据 route meta 解析页面容器 surface，分页 / 表格列表页的 `pageKind: 'list'`
  统一解析为 `paged-table`。
- `meta.pageSurface` 是页面容器宽度的显式覆盖入口，优先级高于 `dashboard` 与 `pageKind` 推导。仅当页面语义与
  默认推导不一致时使用，例如保留 dashboard/tab 语义但需要 `paged-table` 全宽，或保留 list 菜单语义但需要
  `form-detail` 阅读宽度。
- `paged-table` 的内容容器由 `web/src/layouts/components/PageContainer.vue` 与 `web/src/style/layout.less`
  提供全可用宽度，不再使用居中的 `90vw / 1440px` 管理页宽度上限。
- 业务分页页只声明 route/page metadata 与自身表格结构，不应在模块页面内通过局部 max-width、负 margin 或额外
  容器覆盖来修正页面宽度。
- `overview-dashboard` 继续保持居中最大宽度；详情、调查与表单阅读类页面走 `form-detail` 有界宽度。
- 壳层 `PageContainer` 不应在导航开始时直接跟随 live route meta 改宽。全宽页与有界页切换时，离开中的当前页面
  保持原 surface；进入视图在 `out-in` 过渡的 enter 阶段报告目标 surface 后，容器再切换到目标宽度。

非目标：

- 不引入新的 UI 框架
- 不把所有列表页强行做成同一种筛选形态
- 不改变业务 API、权限语义、数据语义
- 不破坏现有 `list-form-detail` 页面模式

## 1. 页面分类

### 1.1 简单分页页 / 轻筛选分页页

适用于筛选字段较少的业务管理列表，例如用户管理、角色管理、权限管理、公告管理、定时任务。

公告管理是当前简单分页页的标准参考：筛选密度、表格信息层级、分页位置可作为样板；但其动作分层仍必须按本文规范收敛，不能保留刷新在 `PageHeader`、列设置在 `FilterBar` 的旧布局。

特征：

- 一个关键字搜索框
- 0 到 3 个常用筛选控件
- 可使用显式查询或即时筛选
- 必须提供列设置
- 可选刷新
- 默认不使用高级筛选器
- 默认不使用快捷筛选，除非业务有强需求

轻筛选页允许两种查询模式：

| 模式 | 适用场景 | 交互规则 |
| --- | --- | --- |
| 显式查询模式 | 服务端分页、查询成本较高、筛选字段较多 | 搜索框 + 筛选控件 + 查询 / 重置 |
| 即时筛选模式 | 本地数据量小、筛选条件简单 | 输入回车、选择变化后查询；保留重置或弱化查询按钮 |

同一类页面的清理文案必须统一，推荐统一为“重置”，不再混用“清空”“清空筛选”“清空条件”。

### 1.2 复杂分页页 / 重筛选分页页

适用于日志、审计、访问记录等排障型页面，例如访问日志、应用日志、审计日志。

特征：

- 关键字搜索框
- 高级筛选器入口
- 快捷筛选 chips
- 当前筛选条件 chips
- 排序 chips
- 查询 / 重置
- 列设置
- 刷新
- 表格说明文案
- 支持较多筛选字段，但默认不撑高首屏

重筛选页默认属于 `query-builder-list-detail` 或 `log-audit` 扩展页型，不应为了统一而牺牲排障效率。

## 3. 页面结构规范

分页 / 列表页统一由以下区域组成：

1. `PageHeader` / 页面标题区
2. `FilterBar` / 筛选区
3. `TableCard` / 表格卡片
4. `Pagination` / 分页区
5. `DetailDrawer` / 详情抽屉，如果页面有详情

### 3.1 页面动作分层

分页 / 列表页动作固定分为三层：

| 层级 | 允许动作 | 禁止动作 |
| --- | --- | --- |
| PageHeader actions | 创建、导出、返回上游上下文、其它不依赖当前表格视图状态的页面动作 | 刷新、列设置、密度设置 |
| FilterBar actions | 查询、重置 | 列设置、刷新、导出 |
| TableCard toolbar actions | 刷新、列设置、密度设置、批量操作 | 创建、返回、筛选查询 |

最终结论：

- PageHeader 右侧只放创建 / 导出 / 返回等页面级动作。
- FilterBar 右侧只放查询 / 重置。
- TableCard 右上角放刷新 / 列设置 / 密度设置。
- 刷新更接近当前表格数据视图动作，必须与列设置一起收口到 TableCard toolbar。
- 如果个别页面没有 TableCard toolbar，应先补 toolbar，不继续把刷新和列设置塞进 PageHeader。

### 3.2 PageHeader 视觉密度

标准管理页可以使用 `PageHeaderCard`，例如用户、角色、权限、公告管理。

日志、审计、访问记录等高频排障页面应优先使用紧凑 PageHeader，避免首屏高度被标题卡片、筛选卡片、表格卡片连续吃掉。

规则：

- 标题说明最多一行。
- 禁止长说明文案撑高标题区。
- 面包屑、所属模块、页面说明不得重复表达同一信息。
- 页面上方已有多标签页导航时，PageHeader 应保持克制。
- 访问日志、应用日志、审计日志当前 PageHeader 均需要压缩。

## 4. 搜索框宽度规范

桌面端搜索框不得因为 `flex: 1` 无意义拉满整行。

| 页面类型 | 桌面宽度 | 推荐 CSS 边界 |
| --- | --- | --- |
| 轻筛选页 | 240px 到 320px | `width: clamp(240px, 28vw, 320px)` |
| 重筛选页 | 360px 到 480px | `width: clamp(360px, 36vw, 480px)` |

补充规则：

- 搜索框与筛选控件之间使用统一 gap，建议沿用页面 spacing token 或 12px。
- 搜索框不应因为右侧按钮而被压缩到低于最小宽度。
- 小屏下搜索框可独占一行。
- 重筛选页第一行不必强行填满宽度，允许留白。
- 日志页关键字搜索框 placeholder 不要过长；长说明放 tooltip 或高级筛选说明里。

可落地建议：

- 将轻筛选搜索宽度沉淀为 `list-search` 类或 CSS 变量，例如 `--graft-list-search-width`。
- 将重筛选搜索宽度沉淀为 `query-search` 类或 CSS 变量，例如 `--graft-query-search-width`。
- 收敛 `AdvancedQueryFilterBuilder` 中关键字搜索的 `flex: 1` 行为，改为固定宽度边界 + 小屏响应式换行。

## 5. 筛选区规范

### 5.1 简单分页页 / 轻筛选页

- 使用单行 `FilterBar`。
- 字段过多时才允许换行。
- 查询、重置固定在 FilterBar 右侧。
- 显式查询模式显示查询 / 重置。
- 即时筛选模式可弱化查询按钮，但仍保留统一的“重置”入口。
- 不允许列设置出现在 FilterBar。
- 不允许刷新出现在 FilterBar。
- 列设置必须存在，并固定在 TableCard toolbar。

推荐模式：

| 页面 | 推荐模式 |
| --- | --- |
| 用户管理 | 即时筛选 |
| 角色管理 | 即时筛选 |
| 权限管理 | 即时筛选 |
| 公告管理 | 显式查询 |
| 公告列表 | 即时筛选 / 弱显式 |
| 通知列表 | 显式查询，状态 tab 即时 |
| 定时任务 | 显式查询 |
| 系统配置 | 按字段复杂度决定 |

### 5.2 复杂分页页 / 重筛选页

默认结构：

1. 第一行：关键字搜索 + 查询 / 重置
2. 第二块：高级筛选器入口
3. 第三行：快捷筛选 chips
4. 第四行：当前筛选条件 chips / 排序 chips

规则：

- 默认状态下筛选区高度不应过高。
- 高级筛选条件默认折叠，只有点击“添加筛选条件”或已有条件时展示。
- 快捷筛选数量过多时横向滚动或折叠，不能撑爆页面。
- 当前访问日志、应用日志的关键字搜索和筛选区高度需要收敛。

## 6. TableCard toolbar 规范

TableCard 顶部统一拆成左右两侧。

左侧：

- 主统计文案：
  - `共 N 条记录`
  - 或 `当前页 N 条 / 共 M 条`
- 副说明文案：
  - 可选
  - 最多一行
  - 日志类可说明排序、保留期、默认时间范围

右侧：

- 刷新
- 列设置
- 密度设置，后续如有

规则：

- 刷新和列设置属于表格视图动作，必须放一起。
- 不允许列设置出现在 FilterBar。
- 不允许刷新在 PageHeader、列设置在 TableCard，造成动作分裂。
- 如果页面没有列设置，可以只放刷新。
- 已有列设置的页面必须迁移到 TableCard toolbar。
- 后续批量操作也归入 TableCard toolbar，并与“已选择 N 条”联动。

## 7. 列设置按钮规范

列设置属于表格视图配置，不属于页面全局动作，也不属于筛选动作。

统一结论：

- 列设置入口放 TableCard 右上角。
- 日志类页面和普通业务页面位置一致。
- 所有分页 / 列表页必须提供列设置。
- 当前已有列设置的页面先迁移位置，保留原有列设置数据结构和本地存储 key。
- 当前没有列设置的页面需要补齐最小列设置能力；只读页也不能省略。

迁移步骤：

1. 为 TableCard 建立标准 toolbar。
2. 将已有列设置入口从 PageHeader / FilterBar 移到 TableCard toolbar。
3. 保留原列设置 localStorage key，避免用户配置丢失。
4. 未实现列设置的分页页补齐最小列设置入口、默认列集合和禁用必选列规则。

## 8. 刷新按钮规范

刷新数据属于当前列表数据视图动作，不再放 PageHeader。

统一结论：

- 所有分页 / 列表页的刷新按钮放 TableCard toolbar 右侧。
- 刷新与列设置相邻。
- PageHeader 不再承载刷新。
- 如果页面没有 TableCard toolbar，先补 toolbar。

## 9. 分页规范

统一规则：

- 分页放 TableCard 底部。
- 左侧保留总数文案。
- 右侧放 TDesign Pagination 控件。
- 默认 pageSize 推荐 20。
- 简单本地管理页可以保留 10，但必须说明为轻量密度例外。
- pageSize options 统一为 `10 / 20 / 50 / 100`。
- 页码、上一页、下一页保持 TDesign 默认行为。
- 首次进入页面默认查询第一页。
- 重置后自动查询第一页。
- 表格少量数据时 footer 保持在 TableCard 底部，不脱离卡片结构。

定时任务补充分页规则：

- 定时任务是典型轻筛选列表管理页，必须纳入列表治理。
- 当前后端 / API 已存在 `limit`、`offset`、`total` 语义时，前端补齐分页，不新增 API。
- UI `current` / `pageSize` 映射为：`limit = pageSize`，`offset = (current - 1) * pageSize`。
- 响应中的 `total`、`limit`、`offset` 用于同步分页状态。
- 不使用 `page` / `page_size` 伪装定时任务契约。

## 10. 表格列规范

- 主列靠左；使用两行结构时副标题弱化。
- 状态列使用 TDesign `Tag`。
- 时间列统一格式。
- `ID`、`request_id`、`event_type`、`resource` 等长文本默认截断，悬浮显示完整值。
- 操作列固定右侧。
- 操作列标题统一为“操作”。
- 操作按钮数量超过 2 个时使用“更多”。
- “详情”优先作为第一个行操作。
- 操作列宽度按操作数量分级，不随单行文案任意抖动。

日志类页面需要重点检查：

- `request_id` 是否截断并提供 tooltip。
- `event_type` 是否有稳定列宽和展示格式。
- `resource` 是否避免撑宽表格。
- 操作列是否固定右侧且标题统一。

## 11. 空态与错误态规范

| 状态 | 展示规则 | 恢复动作 |
| --- | --- | --- |
| 无数据 | 说明该资源当前为空 | 有创建权限时可提供创建入口 |
| 筛选无结果 | 提示“没有符合当前筛选条件的数据” | 提供“重置” |
| 加载失败 | 明确加载失败 | 提供重试按钮 |

补充规则：

- 只读页面的筛选无结果空态只提供恢复筛选动作。
- 创建型页面可以在无数据空态提供创建动作。
- 首次进入页面默认查询第一页。
- 重置后自动查询第一页。
- 错误态不得伪装成普通空态。

## 12. 可复用组件设计

优先抽取布局约束和小组件，不做大而全万能组件。

推荐收敛：

- `ListFilterBar`
  - 只负责布局、gap、按钮槽位、响应式换行
  - 不接管业务筛选状态
- `TableToolbar`
  - 负责主统计、副说明、右侧动作槽
  - 统一刷新、列设置、密度位置
- `ColumnSettingsButton`
  - 统一入口、图标、tooltip、可访问性
  - 不强改已有本地存储 key
- `ListPagination`
  - 包装项目默认 pageSize/options/summary
  - 不改变 TDesign Pagination 行为
- `ManagementPageHeader` compact 变体
  - 供访问日志、应用日志、审计日志使用
- `AdvancedQueryFilterBuilder`
  - 收敛关键字搜索宽度边界
  - 保留高级筛选 authority 与 URL query 语义

不建议：

- 不把每个页面的业务筛选逻辑塞进一个巨大通用组件。
- 不让共享组件拥有业务 API 查询。
- 不让共享组件隐式处理 route query 同步。
- 不创建第二套 Table / Pagination 运行基线。

## 13. 收敛计划

### Phase 1：规范与低风险统一

- 补充本文档作为前端分页 / 列表页规范。
- 统一搜索框宽度 token / class。
- 统一“查询 / 重置 / 刷新 / 列设置”的语义和位置。
- 统一列设置入口位置到 TableCard toolbar。
- 所有分页页补齐列设置入口。
- 统一重置文案为“重置”。
- 日志 / 审计页先压缩 PageHeader 规则，不立即重写查询模型。
- 不改 API，不改业务逻辑。
- 不破坏 URL query 同步和列设置 localStorage key。

### Phase 2：共享布局组件

- 抽取或收敛 `ListFilterBar`、`TableToolbar`、`ColumnSettingsButton`、`ListPagination`。
- 为 `ManagementPageHeader` 增加 compact 模式。
- 为 `AdvancedQueryListPage` 支持 TableCard toolbar 插槽。
- 保持业务页面自己声明筛选字段、columns、查询 API 和 route query 同步。
- 定时任务改用统一 TableCard / toolbar 结构，并接入分页状态。
- 避免大而全的万能列表组件。

### Phase 3：逐页迁移

建议顺序：

1. 迁移 1 个轻筛选样板：角色管理或公告管理。
2. 迁移定时任务：补分页，使用现有 `limit` / `offset`，列设置和刷新进入 TableCard toolbar。
3. 迁移 1 个重筛选样板：访问日志。压缩 PageHeader，固定搜索框宽度，刷新 + 列设置进入 TableCard toolbar。
4. 复用样板迁移应用日志、审计日志。
5. 迁移用户管理、权限管理、公告列表、通知列表。
6. 最后处理系统配置和非典型列表页，只应用动作分层和 toolbar 规则，不强行改造成分页页。

验证覆盖：

- 暗色主题
- 窄屏
- 英文文案
- 列设置
- 分页
- route query 恢复
- 详情抽屉

## 14. 风险点

- URL query 同步：访问日志、应用日志、审计日志、通知列表必须保持现有 deep link 和刷新恢复行为。
- 列设置本地存储 key：例如公告管理 `graft.announcement.management.visibleColumns` 不得因迁移丢失用户配置。
- TDesign Table / Pagination 行为：不要混用 `page/pageSize` 与 `limit/offset`；定时任务使用现有 `limit/offset`。
- 过度抽象：共享组件只承接布局和动作槽，不承接业务查询模型。
- `list-form-detail` 页面结构：轻筛选页仍保持 list-form-detail 模式，不改详情抽屉和表单流。
- 首屏高度：日志 / 审计页如果只搬按钮不压缩 Header，仍无法达到验收。
- 跨模块 shared 提升：新增共享组件前需确认它们无业务语义，符合 `shared/**` 复用边界。

## 15. 验收标准

- 所有分页 / 列表页动作符合三层规则：
  - PageHeader 无刷新 / 列设置
  - FilterBar 无列设置
  - TableCard toolbar 放刷新 / 列设置
- 访问日志、应用日志、审计日志使用紧凑 PageHeader，标题说明不超过一行。
- 轻筛选页明确显式查询或即时筛选模式，同类文案统一为“重置”。
- 搜索框桌面端不再无意义拉满，窄屏可自然换行。
- 定时任务补齐分页，并使用现有 `limit` / `offset` API。
- 表格顶部统计、副说明、toolbar 结构一致。
- 分页区域位置、pageSize options、总数文案一致。
- 日志类长字段有截断和 tooltip，操作列标题统一为“操作”。
- 无数据、筛选无结果、加载失败三类状态可区分。
- 暗色主题下无错位。
- 英文文案不撑爆布局。
- 不改变业务 API、权限、数据语义、route query 语义。
