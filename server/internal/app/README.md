# app

## 用途

`app` 负责装配 `server` 运行时外壳，并按显式顺序驱动核心资源与插件生命周期。

## 职责边界

这个模块负责：

* 加载核心依赖并构造运行时
* 注册核心服务与健康检查路由
* 驱动插件的 `Register`、`Boot`、`Shutdown`
* 在失败路径和正常关闭路径释放核心资源
* 持有并注入 `event bus`、注册表与其它 core 级共享基础设施

这个模块不负责：

* 承载具体业务能力
* 隐式发现插件
* 把插件生命周期隐藏在反射或框架魔法里

## 主要入口

* `doc.go`：包职责说明
* `runtime.go`：运行时装配、核心服务注册与生命周期执行
* `runtime_test.go`：插件关闭顺序与错误聚合验证

## 关键依赖

* 下游依赖 `config`、`database`、`redisx`、`httpx`、`eventbus`、`plugin`、`store` 等核心模块
* 上游由 `server/cmd/graft` 命令入口创建并执行

## 维护提示

这里的价值是让启动顺序和资源归属始终可见。若后续改动让核心装配变成隐式行为，应先回看 `ai-plan/design/项目设计.md`，确认是否破坏了“显式装配、轻量 DI、插件边界清晰”的前提。
