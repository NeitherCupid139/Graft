# container

## 用途

`container` 提供 `server` 运行时使用的显式单例容器。

## 职责边界

这个模块只负责：

* 注册单例服务构造函数
* 按需解析共享实例
* 保证单例构建过程的并发安全

这个模块不负责：

* 反射扫描
* 隐式注入
* 自动装配整张依赖图
* 复杂作用域管理

## 主要入口

* `doc.go`：包职责说明
* `container.go`：容器接口、实现与并发控制
* `container_test.go`：单例并发构建行为验证

## 关键依赖

* 上游由 `server/internal/app` 与插件注册流程调用
* 下游会被核心运行时和插件用作公开服务注册与解析入口

## 维护提示

如果这里的能力开始依赖反射、包扫描或隐藏生命周期，说明实现已经偏离仓库的轻量 DI 约束，应先回看 `ai-plan/design/插件与依赖注入设计.md`。
